¿Qué es el Botón Bancario?

Un medio de pago que carga directo desde la cuenta corriente del cliente — sin tarjeta. El usuario elige su banco, inicia sesión y confirma con clave/2FA. Soporta montos altos; por eso Toku lo prioriza sobre transferencia solo cuando el monto lo amerita, sin pedirle nunca al usuario elegir entre ambos: la orquestación es 100% interna.

Pago cuenta a cuenta
Sin tarjeta de crédito ni débito — el usuario se autentica directo en su banco.
Pensado para montos altos
Es más costoso que una transferencia, por eso Toku lo usa solo sobre cierto monto — la orquestación decide automáticamente.
Confirmación por webhook
El banco notifica a Toku cuando el pago se aprueba. Un pago no exitoso queda "abortado", no "failed" — es normal.
Pagos B2B con doble aprobador
Pueden tardar hasta 8 horas en confirmarse mientras esperan la segunda aprobación — no es un error.
Cómo se implementa del lado de CAT

CAT nunca se conecta directo al banco. Todo su trabajo técnico es consumir 2 endpoints de Toku y mostrar 2 pantallas — el resto (credenciales, firmas, conciliación) lo resuelve Toku.

CAT 1. Inicia el pago El frontend de CAT llama al endpoint de inicio de Toku cuando el usuario elige pagar.
→
Toku 2. Crea la transacción Toku arma la transacción y entrega la URL (o config) del banco elegido.
→
Banco 3. El usuario paga CAT redirige al banco; el usuario inicia sesión y confirma con clave/2FA.
→
Toku 4. Confirma el pago El banco notifica por webhook a Toku, que valida y actualiza el estado.
→
CAT 5. Muestra el resultado El usuario vuelve a CAT y ve éxito, rechazo o pendiente.
Lo que CAT nunca implementa: integración directa con la API del banco, ni el parsing de archivos de rendición/SFTP — eso es 100% Toku.
¿Con los bancos de Toku o los propios de CAT?

La velocidad de lanzamiento depende de esta decisión — y se puede mezclar banco por banco.

PSP — convenio de Toku con el banco Para partir rápido
Ya disponible con Scotiabank, Santander y Banco de Chile — se conecta sin gestionar nada con el banco.
Para bancos donde CAT no tenga convenio propio (por ejemplo Bice o BCI), PSP permite sumarlos sin depender de esa negociación.
CAT no gestiona contratos ni credenciales bancarias.
PST — convenio propio de CAT depende del banco
CAT debe contratar el botón bancario directamente con cada banco (Toku entrega carta tipo y contactos).
El tiempo depende del banco — generalmente semanas.
Las credenciales quedan asociadas a la cuenta de CAT.

Ambos modelos conviven: se puede partir 100% PSP para lanzar rápido, y migrar banco por banco a PST más adelante si conviene.
